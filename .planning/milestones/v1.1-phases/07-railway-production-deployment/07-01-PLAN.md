  <name>Task 1: Fix Dockerfile for PostgreSQL and add startup script</name>
  <files>backend/Dockerfile, backend/docker/start.sh, backend/.dockerignore</files>
  <action>
1. **Fix backend/Dockerfile:**
   - Replace `pdo_mysql` with `pdo_pgsql` on the `docker-php-ext-install` line
   - Add `libpq-dev` to the `apk add` package list (required to compile pdo_pgsql)
   - After the "Set permissions" block, add lines to COPY and chmod the startup script:
     ```
     COPY docker/start.sh /var/www/docker/start.sh
     RUN chmod +x /var/www/docker/start.sh
     ```
   - Replace the final `CMD ["/usr/bin/supervisord", "-c", "/etc/supervisord.conf"]` with:
     ```
     CMD ["/var/www/docker/start.sh"]
     ```

2. **Create backend/docker/start.sh:**
   ```bash
   #!/bin/sh
   set -e

   # Run migrations on every deploy
   php artisan migrate --force

   # Cache configuration for production performance
   php artisan config:cache
   php artisan route:cache
   php artisan view:cache
   php artisan event:cache

   # Start supervisor (PHP-FPM + Nginx)
   exec /usr/bin/supervisord -c /etc/supervisord.conf
   ```

3. **Fix backend/.dockerignore:**
   - Remove the line `docker` from .dockerignore (the Dockerfile COPYs files from docker/ directory -- nginx.conf, supervisord.conf, start.sh)
   - Keep all other entries as-is
  </action>
