<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Create Railway project and deploy services</name>
  <files></files>
  <action>
User must create the Railway project and services via the Railway dashboard. Claude cannot access the Railway dashboard.

**Step-by-step instructions:**

### A. Create Railway Project
1. Go to https://railway.com/dashboard
2. Click "New Project" -> "Empty Project"
3. Name it "aqua-tip"

### B. Add PostgreSQL Database
1. In the project, click "+ New" -> "Database" -> "PostgreSQL"
2. Wait for it to provision (takes ~30 seconds)

### C. Create Backend Service
1. Click "+ New" -> "GitHub Repo"
2. Select the aqua-tip repository
3. In the service settings, set **Root Directory** to `backend`
4. Railway will auto-detect the Dockerfile and start building
5. Go to **Variables** tab and add all variables from `backend/.env.production`:
   - `APP_NAME` = `AQUA TIP`
   - `APP_ENV` = `production`
   - `APP_KEY` = (run `php artisan key:generate --show` in backend/ directory locally, paste the output)
   - `APP_DEBUG` = `false`
   - `APP_URL` = (leave blank for now, update after domain is generated)
   - `FRONTEND_URL` = (leave blank for now, update after frontend domain is generated)
   - `DB_CONNECTION` = `pgsql`
   - `DB_URL` = `${{Postgres.DATABASE_URL}}` (use the Railway reference variable dropdown)
   - `SESSION_DRIVER` = `database`
   - `SESSION_LIFETIME` = `480`
   - `SESSION_ENCRYPT` = `true`
   - `SESSION_SECURE_COOKIE` = `true`
   - `SESSION_SAME_SITE` = `lax`
   - `LOG_CHANNEL` = `stderr`
   - `LOG_LEVEL` = `warning`
   - `CACHE_STORE` = `database`
   - `QUEUE_CONNECTION` = `database`
   - `MAIL_MAILER` = `log`
   - `BCRYPT_ROUNDS` = `12`
6. Go to **Settings** -> **Networking** -> click "Generate Domain" to get a public URL
7. Copy the generated domain (e.g., `backend-xxx.up.railway.app`)
8. Update `APP_URL` variable to `https://backend-xxx.up.railway.app`
9. Trigger a redeploy if needed

### D. Create Frontend Service
1. Click "+ New" -> "GitHub Repo"
2. Select the same aqua-tip repository
3. In the service settings, set **Root Directory** to `frontend`
4. Railway/Nixpacks will auto-detect Node.js
5. In **Settings**, set **Start Command** to `node server.js`
6. Go to **Settings** -> **Networking** -> click "Generate Domain"
7. Copy the frontend domain (e.g., `frontend-yyy.up.railway.app`)

### E. Update Backend Variables with Domains
1. Go back to Backend service -> Variables
2. Set `FRONTEND_URL` = `https://frontend-yyy.up.railway.app`
3. Set `SANCTUM_STATEFUL_DOMAINS` = `frontend-yyy.up.railway.app`
4. Trigger redeploy of backend

### F. Verify Deployments
1. Wait for both services to finish deploying (check build logs)
2. Visit the backend URL -- should see Laravel response (not a 500 error)
3. Visit the frontend URL -- should see the AQUA TIP landing page
4. Check Railway deploy logs for the backend to confirm migrations ran

**When done, provide:**
- Backend URL (e.g., https://backend-xxx.up.railway.app)
- Frontend URL (e.g., https://frontend-yyy.up.railway.app)
- Confirmation that both load without errors
