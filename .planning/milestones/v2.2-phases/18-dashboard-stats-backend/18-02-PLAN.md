**Test cases to write:**

1. `getCounts returns 4 entity type counts from OpenCTI` -- Mock OpenCtiService->query to return `['stixCyberObservables' => ['pageInfo' => ['globalCount' => 42]]]` for each call. Assert result has 4 items, each with entity_type, label, count keys. Assert count values match mocked globalCount.

2. `getCounts returns stale cache when OpenCTI fails` -- Pre-populate cache with `Cache::put('dashboard_counts', $staleData, now()->addMinutes(10))`. Mock OpenCtiService->query to throw `OpenCtiConnectionException`. Call getCounts(). Assert it returns $staleData (the stale cached value).

3. `getCounts throws when OpenCTI fails and no cache exists` -- Clear cache. Mock OpenCtiService->query to throw `OpenCtiConnectionException`. Assert getCounts() throws `OpenCtiConnectionException`.

4. `getIndicators returns normalized observable array` -- Mock OpenCtiService->query to return:
```php
['stixCyberObservables' => ['edges' => [
    ['node' => ['id' => 'x1', 'entity_type' => 'IPv4-Addr', 'observable_value' => '1.2.3.4', 'x_opencti_score' => 75, 'created_at' => '2026-03-19T10:00:00Z']],
]]]
```
Assert result is `[['id' => 'x1', 'value' => '1.2.3.4', 'entity_type' => 'IPv4-Addr', 'score' => 75, 'created_at' => '2026-03-19T10:00:00Z']]`.

5. `getCategories returns top 6 labels sorted by count descending` -- Mock OpenCtiService->query to return 500-node response with various objectLabel values (create 8 distinct labels with different frequencies). Assert result has exactly 6 items. Assert first item has highest count. Assert items are sorted descending by count.

6. `getCategories handles observables with no labels` -- Mock response with nodes that have empty `objectLabel: []`. Assert result is empty array (no crash).

7. `getCounts caches result for 5 minutes` -- Clear cache. Mock OpenCtiService->query. Call getCounts(). Assert `Cache::has('dashboard_counts')` is true.

**Important:** Call `Cache::flush()` in a `beforeEach` to ensure test isolation:

beforeEach(function () {
