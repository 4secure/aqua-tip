**1. Create `backend/app/Http/Controllers/ThreatActor/EnrichmentController.php`** (per D-12):

```php
<?php

namespace App\Http\Controllers\ThreatActor;

use App\Exceptions\OpenCtiConnectionException;
use App\Http\Controllers\Controller;
use App\Services\ThreatActorService;
use Illuminate\Http\JsonResponse;

class EnrichmentController extends Controller
{
    /**
     * Get enrichment data for a single threat actor (intrusion set).
     *
     * GET /api/threat-actors/{id}/enrichment
     */
    public function __invoke(string $id): JsonResponse
    {
        try {
            $data = app(ThreatActorService::class)->enrichment($id);
        } catch (OpenCtiConnectionException) {
            return response()->json([
                'message' => 'Unable to load enrichment data. Please try again.',
            ], 502);
        }

        return response()->json(['data' => $data]);
    }
}
```

**2. Add `enrichment()` method to `ThreatActorService.php`** (per D-13):

Add a new public method `enrichment(string $id): array` that:
- Cache key: `'threat_actor_enrichment:' . $id`
- Cache TTL: 15 minutes (per D-11)
- Calls a private `executeEnrichmentQuery(string $id)` method

The GraphQL query inside `executeEnrichmentQuery` must query `intrusionSet(id: $id)` with these nested `stixCoreRelationships` aliases:

```graphql
query ($id: String!) {
  intrusionSet(id: $id) {
    attackPatterns: stixCoreRelationships(
      relationship_type: "uses"
      toTypes: ["Attack-Pattern"]
      first: 100
    ) {
      edges {
        node {
          to {
            ... on AttackPattern {
              id
              name
              x_mitre_id
              killChainPhases {
                edges {
                  node {
                    kill_chain_name
                    phase_name
                    x_opencti_order
                  }
                }
              }
            }
          }
        }
      }
    }
    tools: stixCoreRelationships(
      relationship_type: "uses"
      toTypes: ["Tool"]
      first: 50
    ) {
      edges {
        node {
          to {
            ... on Tool {
              id
              name
            }
          }
        }
      }
    }
    malware: stixCoreRelationships(
      relationship_type: "uses"
      toTypes: ["Malware"]
      first: 50
    ) {
      edges {
        node {
          to {
            ... on Malware {
              id
              name
            }
          }
        }
      }
    }
    campaigns: stixCoreRelationships(
      relationship_type: "attributed-to"
      fromTypes: ["Campaign"]
      first: 50
    ) {
      edges {
        node {
          from {
            ... on Campaign {
              id
              name
              first_seen
              last_seen
            }
          }
        }
      }
    }
    allRelationships: stixCoreRelationships(first: 100) {
      edges {
        node {
          id
          relationship_type
          from {
            ... on BasicObject {
              id
              entity_type
            }
            ... on StixDomainObject {
              name
            }
            ... on StixCyberObservable {
              observable_value
            }
          }
          to {
            ... on BasicObject {
              id
              entity_type
            }
            ... on StixDomainObject {
              name
            }
            ... on StixCyberObservable {
              observable_value
            }
          }
        }
      }
    }
  }
}
```

**Normalization** - Add a private `normalizeEnrichmentResponse(array $data): array` method that returns:

```php
[
    'ttps' => [...],        // Grouped by tactic
    'tools' => [...],       // Flat array of {id, name, type: 'tool'}
    'malware' => [...],     // Flat array of {id, name, type: 'malware'}
    'campaigns' => [...],   // Flat array of {id, name, first_seen, last_seen}
    'relationships' => [...] // Flat array of {id, relationship_type, from: {id, entity_type, name}, to: {id, entity_type, name}}
]
```

For TTPs normalization:
- Extract attack patterns from `$data['intrusionSet']['attackPatterns']['edges']`
- For each attack pattern, read `killChainPhases.edges[].node` and filter for `kill_chain_name === 'mitre-attack'`
- Group techniques by `phase_name` (the tactic slug like `initial-access`, `execution`)
- Each TTP entry: `['id' => ..., 'name' => ..., 'mitre_id' => $node['x_mitre_id'] ?? null]`
- Each tactic group: `['tactic' => 'initial-access', 'tactic_label' => 'Initial Access', 'techniques' => [...]]`
- Techniques with no mitre-attack kill chain phase go into tactic `'other'` with label `'Other'`
- Sort tactic groups using this order array:
  `['reconnaissance', 'resource-development', 'initial-access', 'execution', 'persistence', 'privilege-escalation', 'defense-evasion', 'credential-access', 'discovery', 'lateral-movement', 'collection', 'command-and-control', 'exfiltration', 'impact']`
- `'other'` group goes last

For tools: extract from `tools.edges[].node.to`, return `['id' => ..., 'name' => ..., 'type' => 'tool']`
For malware: extract from `malware.edges[].node.to`, return `['id' => ..., 'name' => ..., 'type' => 'malware']`
For campaigns: extract from `campaigns.edges[].node.from` (NOTE: `from` not `to`, because Campaign --attributed-to--> IntrusionSet), return `['id' => ..., 'name' => ..., 'first_seen' => ..., 'last_seen' => ...]`
For relationships: extract from `allRelationships.edges[].node`, return flat objects with from/to containing `id`, `entity_type`, `name` (falling back to `observable_value` then `entity_type`)
