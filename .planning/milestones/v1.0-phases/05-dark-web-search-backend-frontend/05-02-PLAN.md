<!-- Backend API contracts from Plan 01 -->

POST /api/dark-web/search
Request: { query: string, type: "email" | "domain" }
Success 200: {
  data: { found: number, results: Array<{ email, password, username, source, breach_date, first_name, last_name, phone, fields }> },
  credits: { remaining: number, limit: number, resets_at: string }
}
Error 429: { message: string, remaining: 0, limit: number, resets_at: string, is_guest: boolean }
Error 502: { message: "Something went wrong. No credit was deducted.", credits: { remaining, limit, resets_at } }
Error 422: { message: string, errors: { query?: string[], type?: string[] } }
Error 401: unauthenticated

GET /api/credits
Response: { remaining: number, limit: number, resets_at: string }

1. Replace `frontend/src/pages/DarkWebPage.jsx` placeholder with full implementation:

   **State management:**
   - `searchType` (email|domain) -- toggle state, default 'email'
   - `query` -- input value
   - `results` -- null (initial), array after search
   - `searchMeta` -- { found, query } from last search for header
   - `credits` -- { remaining, limit, resets_at } fetched on mount
   - `loading` -- boolean
   - `error` -- null or { message, refunded } object
   - `hasSearched` -- boolean for layout transition
   - `recentQueries` -- array from localStorage
   - `showRecent` -- boolean for dropdown visibility

   **On mount (useEffect):**
   - Fetch credits via `fetchCredits()` and set state
   - Load recent queries from localStorage key `darkweb_recent_queries`

   **Search flow:**
   - Client-side validation first: email regex `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` for email, domain regex `/^[a-zA-Z0-9]([a-zA-Z0-9-]*[a-zA-Z0-9])?(\.[a-zA-Z]{2,})+$/` for domain
   - If invalid, set error with validation message, do NOT call API
   - Set loading=true, disable search button and input
   - Call `searchDarkWeb({ query, type: searchType })`
   - On success: set results, update credits from response, add to recent queries (localStorage), set hasSearched=true
   - On 429: set error with message from response, update credits to show 0
   - On 502: set error with "Something went wrong. No credit was deducted." and refunded=true flag
   - On other error: set generic error message
   - Always set loading=false

   **Recent queries:**
   - Store in localStorage key `darkweb_recent_queries` as JSON array of `{ query, type, timestamp }`
   - Max 5 entries, deduplicated by query+type
   - Show dropdown below search input on focus when `recentQueries.length > 0`
   - Each item clickable -- sets query and searchType, then submits
   - Close dropdown on blur (with slight delay for click registration) or on search submit

   **Layout -- initial state (hasSearched=false):**
   - Centered vertically in viewport: `flex flex-col items-center justify-center min-h-[60vh]`
   - Shield/search icon (from Lucide: ShieldAlert or Globe)
   - "Dark Web Search" heading (font-display text-3xl font-bold text-text-primary)
   - Tagline (font-mono text-sm text-text-muted): "Search for breached credentials across known data breaches"
   - Email/Domain toggle: two buttons side by side, active one gets violet bg, inactive gets surface bg
   - Search input + search button + CreditBadge in a row
   - Use Framer Motion `layout` prop on the container div for smooth transition

   **Layout -- results state (hasSearched=true):**
   - Search bar area at top (sticky/fixed-ish with surface bg): toggle + input + button + CreditBadge
   - Results summary: "X breaches found for [query]" or appropriate message
   - If `error` with `refunded=true`: red-tinted card with error message + Retry button (re-submits same query)
   - If `results.length === 0` and no error: green success card "No breaches found for [query]. [query] appears safe in known data breaches."
   - If results: responsive grid of BreachCard components (1 col mobile, 2 cols lg)
   - Use Framer Motion `AnimatePresence` for results appearing

   **Credits exhausted state:**
   - When `credits.remaining === 0`: search input and button disabled/grayed
   - Inline message below search: "Daily limit reached. Your credits reset at 00:00 UTC." in amber text
   - CreditBadge shows 0/X in red

   **Loading state:**
   - Skeleton cards or spinner in results area while loading
   - Search button shows spinner icon, disabled
