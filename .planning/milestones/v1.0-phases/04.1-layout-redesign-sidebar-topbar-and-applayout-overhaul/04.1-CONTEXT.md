# Phase 4.1: Layout Redesign — Sidebar, Topbar, and AppLayout overhaul - Context

**Gathered:** 2026-03-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Redesign the sidebar navigation, topbar, and AppLayout shell. Includes new nav structure, collapsible sidebar, mobile responsive behavior, glassmorphism styling, auth-aware nav states (disabled items for guests), and fixing public access to IP Search page. Does NOT include building the new pages (Dark Web, Threat Actors, Threat News) — only the nav items pointing to them.

</domain>

<decisions>
## Implementation Decisions

### Sidebar Navigation Items
- Flat list with NO group headers — remove all 4 current groups
- New nav items in order:
  1. Dashboard
  2. IP Search (renamed from IOC Search — the old "IP Lookup" page is removed)
  3. Threat Map
  4. Dark Web (new page placeholder — will have its own search bar, built in a future phase)
  5. Threat Actors (new page placeholder)
  6. Threat News (new page placeholder)
  7. Settings (collapsible submenu containing "Profile Management")
- Removed items: IP Lookup, Threat Feeds, Vuln Scanner, Domain Intel, CVE Explorer, Components

### Auth-Aware Nav States
- **Unauthenticated users:** All nav items visible but disabled/grayed out with a lock icon — EXCEPT IP Search which is fully active
- **Settings menu:** Hidden entirely for unauthenticated users
- **Clicking a locked item:** Redirects to /login
- **Authenticated users:** All items enabled, Settings visible with collapsible submenu

### Sidebar Collapse/Expand
- Collapsible to ~64px icon-only rail via toggle button inside the sidebar
- Hovering over collapsed sidebar temporarily expands it as an overlay (shows labels)
- User collapse preference persisted (localStorage)
- Logo adapts: full logo when expanded, icon-only when collapsed

### Mobile Responsive
- On small screens, sidebar becomes an overlay drawer (hidden by default)
- Hamburger button in topbar to open sidebar drawer on mobile
- Topbar stays visible at all screen sizes

### Topbar Content
- Keep: Breadcrumb ("AquaSecure / Page Name"), PRO badge, Notification bell, Avatar
- Remove: Search trigger (Cmd+K modal) — removed entirely, no relocation
- Avatar dropdown in topbar with: "Manage Profile" + "Logout"
- Unauthenticated: Show "Log In" (ghost) + "Sign Up" (gradient) buttons

### Landing Page → App Navigation
- Landing page CTAs point to /ip-search (not /login)
- Fix: IP Search page must be publicly accessible (currently ProtectedRoute blocks guests)
- Guests see the full AppLayout (sidebar with locked items + topbar with login/signup buttons)

### Logo/Branding
- Replace the shield SVG icon with the actual logo.png from public/
- Keep "AquaSecure" text and "Threat Intel Platform" subtitle

### Claude's Discretion
- Exact collapsed sidebar width and animation timing
- Hamburger icon placement and mobile drawer animation
- SearchModal and NotificationDrawer component cleanup (SearchModal removed)
- Placeholder page content for Dark Web, Threat Actors, Threat News
- Settings submenu expand/collapse animation
- Breakpoint for mobile vs desktop layout switch

</decisions>
