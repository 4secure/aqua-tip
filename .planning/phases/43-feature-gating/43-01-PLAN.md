  <behavior>
    - Free-plan user (plan slug "free", no active trial) hitting any gated endpoint returns 403 with JSON `{ "error": "upgrade_required", "message": "Upgrade your plan to access this feature" }`
    - Basic-plan user hitting gated endpoint returns 200 (passes through)
    - Pro-plan user hitting gated endpoint returns 200
    - Enterprise-plan user hitting gated endpoint returns 200
    - Trial user (plan_id null, trial_ends_at in future) hitting gated endpoint returns 200
    - Expired trial user assigned to free plan returns 403
    - Unauthenticated request passes through (let auth middleware handle it)
  </behavior>
  <action>
    1. Create `backend/app/Http/Middleware/FeatureGate.php` (per D-13, D-15, D-16):
       - Namespace: `App\Http\Middleware`
       - `handle(Request $request, Closure $next): Response`
       - If no user (`$request->user() === null`), pass through (auth middleware handles rejection)
       - Check trial: if `$user->trial_ends_at !== null && $user->trial_ends_at->isFuture() && $user->plan_id === null`, pass through (per D-04 — trial = full access)
       - Load plan relationship: `$user->loadMissing('plan')`
       - If `$user->plan !== null && $user->plan->slug === 'free'`, return 403 JSON response with `error: "upgrade_required"` and `message: "Upgrade your plan to access this feature"`
       - Otherwise (paid plans, no plan assigned), pass through
       - Do NOT inject CreditResolver — this middleware only checks plan slug, no credit logic

    2. Register middleware alias in `backend/bootstrap/app.php`:
       - Add `'feature-gate' => \App\Http\Middleware\FeatureGate::class` to the existing `$middleware->alias([...])` array alongside `deduct-credit`

    3. Create `backend/tests/Feature/FeatureGate/FeatureGateMiddlewareTest.php` with Pest syntax:
       - Use the `/threat-actors` endpoint as the test target (it will be feature-gated in Task 2)
       - Test: free-plan user gets 403 with correct JSON shape (`error`, `message` fields)
       - Test: basic-plan user gets 200
       - Test: pro-plan user gets 200
       - Test: enterprise-plan user gets 200
       - Test: trial user (plan_id null, trial_ends_at 30 days future) gets 200
       - Test: expired trial on free plan gets 403
       - Use existing test patterns: create User with Plan association, `actingAs($user)`
       - Note: the tests for the gated route will only pass after Task 2 applies the middleware to routes. Write the middleware and alias first, then proceed to Task 2 which applies it. The tests should be written to test against the final route configuration.
