  <action>
The ip-api.com free tier does NOT support HTTPS -- only their paid Pro plan does. Replace all three calls with ipapi.co which provides a free HTTPS tier (1000 req/day, no API key needed).

**Important:** The ipapi.co response format differs from ip-api.com. Map fields accordingly:
- ip-api.com `countryCode` -> ipapi.co `country_code`
- ip-api.com `regionName` -> ipapi.co `region`
- ip-api.com `as` -> ipapi.co `asn`
- ip-api.com `asname` -> ipapi.co `org`
- ip-api.com status check `$body['status'] === 'success'` -> ipapi.co has no status field; check that `$body['ip']` exists and no `'error'` key
- ip-api.com `lon` -> ipapi.co `longitude`
- ip-api.com `lat` -> ipapi.co `latitude`

**File 1: backend/app/Services/IpSearchService.php** (around line 359)
Replace the `fetchGeoFromIpApi` method body. Change:
```php
$response = Http::timeout(5)
    ->get("http://ip-api.com/json/{$ip}", [
        'fields' => 'status,message,country,countryCode,city,regionName,lat,lon,isp,org,as,asname',
    ]);
```
To:
```php
$response = Http::timeout(5)
    ->get("https://ipapi.co/{$ip}/json/");
```
Then update the response parsing. Remove the `status === 'success'` check. Instead check `!isset($body['error'])`. Map the return array:
```php
if (! $response->successful()) {
    return null;
}

$body = $response->json();

if (isset($body['error'])) {
    return null;
}

return [
    'country' => $body['country_name'] ?? null,
    'country_code' => $body['country_code'] ?? null,
    'city' => $body['city'] ?? null,
    'region' => $body['region'] ?? null,
    'lat' => $body['latitude'] ?? null,
    'lon' => $body['longitude'] ?? null,
    'isp' => $body['org'] ?? null,
    'org' => $body['org'] ?? null,
    'as' => $body['asn'] ?? null,
    'asname' => $body['org'] ?? null,
];
```
