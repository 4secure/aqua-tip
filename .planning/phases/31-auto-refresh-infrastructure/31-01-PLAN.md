Create `frontend/src/hooks/useAutoRefresh.js` with a named export `useAutoRefresh(fetchFn, intervalMs)`.

**Hook signature** (per D-03):
```javascript
export function useAutoRefresh(fetchFn, intervalMs = 5 * 60 * 1000)
```

**Return value:** None (fire-and-forget timer per D-04 -- no manual refresh button).

**Implementation requirements:**

1. **Ref for latest fetchFn** -- Store `fetchFn` in a ref that updates on every render. This prevents the interval timer from restarting when `fetchFn` changes (e.g., when search params change). The interval always calls `fetchRef.current`:
   ```javascript
   const fetchRef = useRef(fetchFn);
   useEffect(() => { fetchRef.current = fetchFn; }, [fetchFn]);
   ```

2. **Main useEffect** with `[intervalMs]` dependency array containing:
   - `let intervalId = null;`
   - `startInterval()` function: clears any existing interval via `if (intervalId) clearInterval(intervalId)`, then sets `intervalId = setInterval(...)` that calls `fetchRef.current().catch(() => {})` (per D-07 -- silent error swallowing, keep stale data)
   - `handleVisibilityChange()` function:
     - If `document.hidden` is true (per D-05): clear interval, set `intervalId = null`
     - If `document.hidden` is false (per D-06): call `fetchRef.current().catch(() => {})` immediately, then call `startInterval()` to restart the 5-min timer from that point
   - Call `startInterval()` to begin the timer
   - Add `document.addEventListener('visibilitychange', handleVisibilityChange)`
   - Cleanup: `clearInterval(intervalId)` and `document.removeEventListener('visibilitychange', handleVisibilityChange)`

3. **Imports:** `import { useEffect, useRef } from 'react';`

**Critical details:**
- The interval callback MUST call `fetchRef.current()` (not `fetchFn` directly) to avoid stale closures
- The `.catch(() => {})` on every fetch call ensures D-07 compliance (silent failure)
- `handleVisibilityChange` MUST clear the interval BEFORE doing the immediate fetch on tab return to prevent race condition (Pitfall 5 from RESEARCH.md)
- No visual state of any kind (per D-02) -- no `isActive` return, no timestamps
