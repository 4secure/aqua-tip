1. Create migration `backend/database/migrations/2026_03_21_000002_add_plan_columns_to_users.php`:
   ```php
   Schema::table('users', function (Blueprint $table) {
       $table->foreignId('plan_id')->nullable()->after('trial_ends_at')->constrained('plans')->nullOnDelete();
       $table->string('timezone', 100)->nullable()->after('plan_id');
       $table->string('organization', 255)->nullable()->after('timezone');
       $table->string('role', 255)->nullable()->after('organization');
   });
   ```
   - `down()` method drops columns in reverse order:
   ```php
   Schema::table('users', function (Blueprint $table) {
       $table->dropForeign(['plan_id']);
       $table->dropColumn(['role', 'organization', 'timezone', 'plan_id']);
   });
   ```
   - Uses `nullOnDelete()` so deleting a plan sets user's plan_id to null (not cascade delete)
   - All columns nullable per locked decision
   - `after()` calls for clean column ordering (ignored by SQLite in tests, works in PostgreSQL)
