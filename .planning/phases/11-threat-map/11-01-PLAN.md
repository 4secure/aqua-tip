    Create `backend/app/Data/CountryCentroids.php` with a static `CENTROIDS` array mapping ISO 3166-1 alpha-2 country codes to [lat, lng] pairs. Include at least the top 50 countries by threat activity (US, RU, CN, DE, GB, FR, JP, IN, BR, AU, KR, NL, UA, IR, PK, etc.). Static method `get(string $countryCode): ?array` returns `['lat' => float, 'lng' => float]` or null.

    Create `backend/app/Services/ThreatMapService.php` that does NOT inject OpenCtiService (the stream relay in the controller handles the HTTP connection directly). The service focuses on:

    1. `parseStixEvent(array $raw): ?array` — Extract from raw OpenCTI SSE event data:
       - Extract IP from STIX object: direct `value` for ipv4-addr/ipv6-addr types, regex on `pattern` field for indicators, null otherwise
       - Classify attack type from labels array first (match against known types), then fall back to entity_type mapping
       - Return normalized event: `['id', 'type', 'color', 'ip', 'entity_type', 'timestamp', 'message']` or null if no IP

    2. `resolveGeo(string $ip): ?array` — Geocode IP address:
       - Check cache first: `Cache::remember('geo:' . md5($ip), now()->addHours(1), ...)`
       - Try ip-api.com: `Http::timeout(3)->get("http://ip-api.com/json/{$ip}")`
       - If ip-api returns success: return `['lat', 'lng', 'city', 'country', 'countryCode']`
       - If ip-api fails: try `CountryCentroids::get($countryCode)` if country code is available from other data
       - Return null if all resolution fails

    3. `getSnapshot(): array` — Fetch recent events for initial page load:
       - Use `Cache::remember('threat_map:snapshot', now()->addMinutes(15), ...)`
       - Query OpenCTI via `OpenCtiService::query()` for recent STIX sightings/indicators (last 15 min by created_at filter)
       - Parse each result through parseStixEvent + resolveGeo
       - Return array of geo-annotated events

       - Red: command-and-control, ddos, malware
       - Amber: scanning, business-email-compromise
       - Violet: phishing, apt
       - Cyan: reconnaissance
       - Default: cyan for unknown

  <action>
    Create `backend/app/Http/Controllers/ThreatMap/SnapshotController.php`:
    - Invokable controller following ThreatNews/IndexController pattern
    - Calls `ThreatMapService::getSnapshot()` to get cached recent events
    - Catches OpenCtiConnectionException, returns 502 with error message
    - Returns JSON: `{ data: { events: [...], counters: { threats: N, countries: N, types: N } } }`
    - Counters are computed from the snapshot events (count unique countries, unique attack types)

    Create `backend/app/Http/Controllers/ThreatMap/StreamController.php`:
    - Invokable controller that returns `response()->stream(...)` with SSE headers
    - SSE headers: Content-Type: text/event-stream, Cache-Control: no-cache, Connection: keep-alive, X-Accel-Buffering: no
    - Inside stream closure:
      1. Disable output buffering: `while (ob_get_level()) { ob_end_flush(); }`, set_time_limit(0)
      2. Connect to OpenCTI SSE: `Http::withHeaders(['Authorization' => 'Bearer ' . config('services.opencti.token')])->withOptions(['stream' => true, 'timeout' => 0])->get(config('services.opencti.url') . '/stream')`
      3. Read lines from response body. When line starts with `data: `, parse JSON, call `ThreatMapService::parseStixEvent()`
      4. If event is valid (has IP), call `ThreatMapService::resolveGeo()` to add coordinates
      5. Echo `"retry: 10000\n"` at start (sets browser reconnect interval to 10s per CONTEXT.md)
      6. Echo `"data: " . json_encode($enrichedEvent) . "\n\n"` and flush
      7. Send heartbeat comment (`: heartbeat\n\n`) every 30 seconds to keep connection alive
      8. Check `connection_aborted()` in loop to stop when client disconnects

    Update `backend/routes/api.php`:
    - Add use statements for both controllers
    - Inside the existing `auth:sanctum` middleware group, add:
      ```php
      // Threat map (live streaming, no credit gating)
      Route::get('/threat-map/snapshot', ThreatMapSnapshotController::class);
      Route::get('/threat-map/stream', ThreatMapStreamController::class);
      ```

    - `StreamTest.php`: Test unauthenticated returns 401, test authenticated returns correct SSE headers (Content-Type: text/event-stream). Note: full streaming behavior is hard to test in PHPUnit, so focus on auth and headers.
